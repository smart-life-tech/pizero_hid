# Pi Zero 2 W HID Composite Image Builder

This workspace builds a **Raspberry Pi OS Lite 64-bit** image (`.img`) that:

- boots as a USB HID composite gadget (keyboard + mouse)
- has SSH enabled
- auto-connects to your Wi-Fi

## What this creates

The build process uses [`pi-gen`](https://github.com/RPi-Distro/pi-gen) plus a custom stage in this repo.

Output image is generated by pi-gen in:

- `pi-gen/deploy/`

## Prerequisites

- Linux environment (recommended: WSL2 Ubuntu on Windows)
- `git`, `rsync`, `xz-utils`, `zip`, `dosfstools`
- Docker (recommended) **or** native pi-gen dependencies

For native builds on Debian/Ubuntu/Raspberry Pi OS, install:

```bash
sudo apt-get update
sudo apt-get install -y quilt qemu-user-binfmt debootstrap zerofree libarchive-tools xxd pigz arch-test
```

If `xxd` is unavailable as a standalone package on your distro, install `vim-common` instead.
If `qemu-user-binfmt` is unavailable, install `qemu-user-static binfmt-support`.

## Quick start (WSL/Linux)

1. Open terminal in this repo.
2. Run:

```bash
chmod +x ./build-image.sh
WIFI_SSID="ember_wolf" \
WIFI_PSK="YOUR_WIFI_PASSWORD" \
WIFI_COUNTRY="US" \
./build-image.sh
```

If successful, your final image will be under `pi-gen/deploy/`.

## Flash and use

1. Flash the generated `.img` to a microSD card.
2. Insert into Pi Zero 2 W.
3. Connect **USB data cable to the Pi's USB (OTG) port**, not power-only port.
4. Boot the Pi.
5. On Windows host, it should enumerate as HID keyboard + HID mouse.
6. SSH into the Pi over Wi-Fi.
7. Run post-boot verification:

```bash
sudo /usr/local/sbin/hid-postboot-verify.sh
```

## Notes

- This setup only ensures valid HID enumeration and device nodes (`/dev/hidg0`, `/dev/hidg1`).
- It does **not** auto-send keystrokes/mouse movement by default.
- You can later write your own sender app using those HID device nodes.
